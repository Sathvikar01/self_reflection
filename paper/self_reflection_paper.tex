\begin{abstract}
Large language models (LLMs) often struggle with complex multi-step reasoning, producing plausible yet incorrect reasoning chains. We present a comprehensive empirical study comparing three self-reflection approaches: (1) Fixed-depth self-reflection, (2) Adaptive self-reflection with dynamic depth adjustment, and (3) RL-guided self-reflection using Monte Carlo Tree Search (MCTS). Our experiments on StrategyQA demonstrate that fixed-depth self-reflection achieves the highest accuracy (82.5\%), representing a 40 percentage point improvement over baseline (42.5\%), with statistical significance at $p = 0.0014$. The adaptive method provides a favorable accuracy-efficiency trade-off, achieving 70.0\% accuracy while reducing API calls by 31\%. The RL-guided approach shows task-dependent effectiveness, excelling on complex multi-step reasoning but providing no benefit on simpler tasks. Our analysis provides practical guidelines for deploying self-reflection methods in production systems, balancing accuracy gains against computational costs.
\end{abstract}

\section{Introduction}

Large language models have demonstrated remarkable capabilities across diverse tasks, from natural language understanding to code generation. However, they remain prone to generating plausible-sounding but logically flawed reasoning chains, particularly for problems requiring multi-step inference \cite{wei2022cot}. When an LLM makes an early reasoning error, it often compounds through subsequent steps, leading to confidently wrong answers.

Recent approaches like Chain-of-Thought (CoT) prompting \cite{wei2022cot} and Tree-of-Thought (ToT) search \cite{yao2023tot} have shown promise but lack systematic mechanisms for error detection and correction. Self-reflection---where a model critiques and corrects its own reasoning---offers a promising direction for improving reasoning quality.

In this paper, we present a comprehensive empirical comparison of three self-reflection methods:

\begin{enumerate}
    \item \textbf{Fixed-Depth Self-Reflection}: A consistent reflection depth across all problems
    \item \textbf{Adaptive Self-Reflection}: Dynamic depth adjustment based on problem complexity
    \item \textbf{RL-Guided Self-Reflection}: MCTS-based exploration with process reward evaluation
\end{enumerate}

Our experiments on StrategyQA reveal important trade-offs between accuracy, computational cost, and deployment complexity. We provide statistical significance testing and practical recommendations for deploying these methods in production systems.

\section{Related Work}

\subsection{Chain-of-Thought Reasoning}

Wei et al. \cite{wei2022cot} demonstrated that prompting LLMs to generate intermediate reasoning steps significantly improves performance on arithmetic, commonsense, and symbolic reasoning tasks. However, CoT lacks mechanisms for detecting and correcting errors mid-reasoning.

\subsection{Tree-of-Thought Search}

Yao et al. \cite{yao2023tot} extended CoT by exploring multiple reasoning paths and selecting the best through search. While effective, ToT does not incorporate explicit self-critique or dynamic depth adjustment based on problem characteristics.

\subsection{Process Reward Models}

Lightman et al. \cite{lightman2023verify} showed that step-level supervision improves reasoning verification. We build on this by using LLM-as-Judge for process evaluation, enabling real-time error detection without requiring trained reward models.

\subsection{Reinforcement Learning for Reasoning}

Recent work has explored RL for language generation, primarily for policy optimization during training. Our work applies RL principles at test time, using MCTS for action selection in a reasoning state space.

\section{Methods}

\subsection{Problem Formulation}

We formulate reasoning as a Markov Decision Process $(\mathcal{S}, \mathcal{A}, \mathcal{T}, \mathcal{R})$ where:
\begin{itemize}
    \item $\mathcal{S}$: States are partial reasoning chains
    \item $\mathcal{A}$: Actions are \{Expand, Reflect, Backtrack, Conclude\}
    \item $\mathcal{T}$: Transition function generates next state given current state and action
    \item $\mathcal{R}$: Reward from Process Reward Model (PRM) evaluation
\end{itemize}

\subsection{Fixed-Depth Self-Reflection}

The fixed-depth approach applies a consistent number of reflection iterations (default: 2) to all problems:

\begin{enumerate}
    \item Generate initial reasoning (3 steps minimum)
    \item Classify problem type (factual/reasoning/strategic)
    \item Self-reflect on reasoning to identify flaws
    \item Apply corrections if issues found
    \item Final self-critique before answer generation
\end{enumerate}

\textbf{Key Features}:
\begin{itemize}
    \item Consistent 2 reflection iterations
    \item Knowledge retrieval for factual questions
    \item Early stopping when no issues detected
    \item Final verification before answering
\end{itemize}

\subsection{Adaptive Self-Reflection}

The adaptive method dynamically adjusts reflection depth based on problem complexity:

\textbf{Complexity Analysis Factors}:
\begin{itemize}
    \item Question type (factual/reasoning/strategic)
    \item Complexity markers (negation, multi-part questions)
    \item Query length
\end{itemize}

\textbf{Dynamic Depth Adjustment}:
\begin{align}
\text{Depth} = 
\begin{cases}
1 & \text{if complexity} < 0.3 \\
2 & \text{if } 0.3 \leq \text{complexity} < 0.5 \\
3 & \text{if } 0.5 \leq \text{complexity} < 0.7 \\
4 & \text{if complexity} \geq 0.7
\end{cases}
\end{align}

\textbf{Rollback Mechanism}: Tracks confidence at each step and reverts to the best checkpoint if confidence drops more than 10\%, preventing degradation from over-correction.

\textbf{Overfitting Prevention}: Implements cross-validation with 3 samples and applies majority voting when variance exceeds 0.2.

\subsection{RL-Guided Self-Reflection}

The RL-guided approach uses Monte Carlo Tree Search with Upper Confidence Bound (UCB1) for node selection:

\begin{equation}
\text{UCB1}(s, a) = Q(s,a) + c \sqrt{\frac{\ln N(s)}{N(s,a)}}
\end{equation}

where $Q(s,a)$ is the average reward, $N(s)$ is the visit count, and $c = 1.414$ is the exploration constant.

\textbf{Action Design}:

\begin{table}[h]
\centering
\begin{tabular}{lll}
\toprule
\textbf{Action} & \textbf{Trigger} & \textbf{Effect} \\
\midrule
Expand & Default & Generate next reasoning step \\
Reflect & Score $< 0.2$ & Critique previous step \\
Backtrack & Score $< 0.3$ & Return to higher-scoring node \\
Conclude & Score $> 0.85$ & Generate final answer \\
\bottomrule
\end{tabular}
\caption{Action types and trigger conditions}
\end{table}

\textbf{Process Reward Model}: We use Llama-3.3-70B to evaluate reasoning steps on a scale from $-1$ (incorrect) to $+1$ (correct), enabling targeted error detection.

\section{Experimental Setup}

\subsection{Datasets}

We evaluate on StrategyQA \cite{geva2021strategyqa}, a dataset of multi-step reasoning questions requiring implicit knowledge:

\begin{itemize}

\end{itemize}

\subsection{Statistical Significance}

We performed McNemar's test to assess statistical significance:

\begin{itemize}
    \item \textbf{Baseline vs Fixed}: $\chi^2 = 10.23$, $p = 0.0014$ (highly significant)
    \item \textbf{Baseline vs Adaptive}: $p = 0.0269$ (significant)
    \item \textbf{Fixed vs Adaptive}: $p = 0.6831$ (not significantly different)
\end{itemize}

\textbf{Confusion Analysis} (Fixed vs Baseline):
\begin{itemize}
    \item Both correct: 14 problems
    \item Both wrong: 4 problems
    \item Baseline only correct: 3 problems
    \item Self-reflection only correct: \textbf{19 problems}
\end{itemize}

Self-reflection recovered 19 answers that baseline missed, with only 3 regressions.

\subsection{Why Fixed-Depth Performs Best}

The fixed-depth approach achieves highest accuracy because:
\begin{enumerate}
    \item Consistent reflection depth ensures thorough self-critique
    \item Knowledge retrieval addresses factual knowledge gaps
    \item Early stopping prevents over-reflection on simple problems
    \item Final verification catches remaining errors
\end{enumerate}

\subsection{When to Use Each Method}

\textbf{Fixed-Depth}: Use when maximum accuracy is critical and API costs are not a primary concern (e.g., high-stakes decision support).

\textbf{Adaptive}: Use when scaling to large datasets where cost efficiency matters (e.g., batch processing thousands of queries).

\textbf{RL-Guided}: Use selectively for complex multi-step reasoning tasks where exploration provides clear benefits (e.g., scientific hypothesis generation).

\section{Limitations}

\begin{enumerate}
    \item \textbf{Dataset Size}: Limited to 40-10 problems per benchmark; larger evaluations needed
    \item \textbf{Model Dependency}: Results specific to Llama models; generalization to other models uncertain
    \item \textbf{Computational Cost}: All self-reflection methods significantly increase token usage (6-25x)
    \item \textbf{Strategic Reasoning}: All methods struggle with game strategy and puzzle problems
\end{enumerate}

\section{Conclusion}

We presented a comprehensive comparison of three self-reflection methods for LLM reasoning. Our key findings:

\begin{enumerate}
    \item \textbf{Fixed-depth self-reflection achieves highest accuracy} (82.5\%), with statistically significant improvement over baseline ($p = 0.0014$)
    \item \textbf{Adaptive reflection provides best efficiency} (31\% cost reduction) while maintaining significant accuracy gains
    \item \textbf{RL-guided reflection shows task-dependent effectiveness}, excelling on complex reasoning but providing no benefit on simpler tasks
\end{enumerate}

These results provide practical guidelines for deploying self-reflection in production systems. Future work should explore hybrid approaches that combine the consistency of fixed-depth reflection with the efficiency of adaptive methods.
